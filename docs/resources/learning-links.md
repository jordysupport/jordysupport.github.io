## Research habits

For factual and current work:

1. prefer primary sources;
2. open the underlying source, not only a search snippet;
3. record publication date and the date of the event or data;
4. distinguish source claims from verified facts;
5. show disagreement and uncertainty;
6. revisit time-sensitive claims before publishing or deciding.

## How to judge a tutorial

Before following a video, post, or repository, check:

- publication date and last update;
- exact product/version used;
- whether the creator links official documentation;
- permissions requested;
- whether credentials are handled safely;
- whether the demo uses test data;
- whether failure and rollback are shown;
- whether a copied command downloads and executes code;
- whether comments report current breakage;
- whether the tutorial distinguishes a prototype from production guidance.

A clear tutorial can still become outdated. Verify the current official instructions at the moment you use it.
