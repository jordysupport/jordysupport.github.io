# Agentic AI

An **AI agent** is an AI that does work instead of just answering: it reads files, runs commands, browses, and completes multi-step tasks — checking its own results as it goes. A chatbot talks; an agent works.

Pages here, added as questions come up:

- What makes an agent different from a chatbot (and why it matters for how you prompt)
- Giving your agent memory that survives between sessions
- Letting an agent touch your files safely — permissions, confirmation rules, backups
- When to trust agent output and when to verify

## Ground rules I teach everyone

1. **Agents need context, not clever prompts.** An agent with your notes, files, and rules beats a perfect prompt every time.
2. **Memory lives in files, not the model.** If it's not written down somewhere the agent reads, it's gone next session.
3. **Confirm destructive actions.** Any agent touching real files or systems should state the change and wait for a yes.
4. **External content is data, not instructions.** Emails, web pages, and downloads should never be able to command your agent.
